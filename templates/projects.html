Customer Offer Recommendation


A offer recommendation pipeline built on retails data based on the purchases made using knowledge graph embeddings. Accepted as one of the 24 workshop papers to be presented at SIGIR eCom 2019, Paris, France

Overview
  Companies send a lot of promotional offers and coupons to customers to attract them to buy more. Offer recommendation systems can help to identify relevant offers to users. In this paper, we present a Neural Factorization (NF) model for the task of Offer recommendation. We represent users and offers with Knowledge Graph Embeddings (KGE). Specifically, we model the available data in the form of a Knowledge Graph (KG) and learn embeddings for entities and relations using a standard KGE technique called TransE. We also incorporate the user temporal features in the NF model using Long Short Term Memory (LSTM) with attention framework. We experiment with Kaggle Acquire Valued Shoppers Challenge dataset and show that the performance of our model is significantly better than tree-based methods.

CHATTERJI: Messenger with real time emoji prediciton😄


A messenger platform developed within 24 hours of hackathon for real time chatting with emoji prediction.

Overview
  For the program, we developed the webapp named Chatterji, the Chatting-with-Emoji. My two teammates developed the frontend (in Angular) and the webapp was built on Django framework. I worked on developing the prediction model for the app. We took sentimental analysis, which aims at maximum of 3 classifications (positive, neutral and negative), to a whole new level of 20 classifications (which were basically the 20 emojis our model predicted). Some of these 20 emojis were 😂, 😀, 😁, 😍, 🤗, ❤️, 😭, 💪, 😜, 💙, 😎, 🔥, 😘 and 💯. Although, if I had developed this project in 2020, 😷 would definitely be there.
  The model was a developed on keras,using a RNN (Recurrent Neural Network) composed of 2 LSTM (Long Short Term Memory) layers. After replacing rare words, performing word embeddings using indexing for all the remaining words and padding the tweets, indexed tweets were created and fed into the model to train. It was trained using 5.8 million tweets from Twitter on Google Colab, for the GPU provided. With 25 minutes of training time, our prediction model was ready with pretty decent accuracy of 60.7% on validation set and 41% on test set, compared to the state-of-the-art method for the same. It was pretty easily and effectively able to predict the emojis for short texts (similar to the tweets) as we trained in just on a single corpus. Hope to build a better model in the future that can predict more number of emojis much easily.

Early warning system for site workers


A real time health monitoring system integrated with a prediction and alerting mechanism using real time data

Overview
  Morbidity rate in iron and steel industries is unexpectedly high, about 60% of total workers are morbid workers to go with the figures. Among them, 80% suffered due to respiratory ailments and 17% suffer casualties due to heat stress. In order to tackle this grave a situation, I worked on an system, that can predict such casualties before hand and report the same to the concerned authority, say the responsible manager for those workers. An Android Application was developed, which was to notify the person. The workers were supposed to wear a Fitbit watch which will keep track of their heart beat rate and send the same continuously to the servers. Apart from that, devices that can supply real time WBGT (Wet-Bulb Globe Temperature) and RPSM(Respirable Suspended Particulate Matter) value of the room. Data from all these devices was sent to MySQL database from where the application would fetch data and predict if a worker is in bad shape or needs immediate attention. Some of the parameters considered for prediction were worker id, age of worker, has drinking and smoking habits, area of work, type of work , WBGT value, Heart rate, RSPM value etc. A multi-label SVM classifier was used, with hyperparameter tuning using Genetic Algorithm, was used to predict the stability condition for workers as good(green), the health situation might go worse(yellow), needs immediate attention(red). Later, in order to automate the process, OpenCV for multi-scale template matching, was used in the main computer system.

Decision Support System for occupational safety


A GUI based system to make risk prediction based on occupational safetly data.

Overview
  Every year, the deaths caused due to lack of occupational safety alone is gruesome to know. In order to alleviate this grave situation, we tried to develop a Decision Support System (DSS) in occupational safety. The system was completely built using Python. For developing the user interface, we used Kivy. Using the system, we can perform Descriptive Analysis and Predictive Analysis with Evaluation of Models based performance parameters like accuracy, precision, recall, F-1 score and AUC value and ROC curve and automatic selection of the model is done based on the best parameters' values. The hyperparameters of model were tuned using Genetic Algorithms. The system can prove to be immensely useful for occupational safety front for industries, reducing casualties.
